# Security Checklist

## Data Protection

- [ ] Sensitive fields identified and documented
- [ ] Data classification applied (public / internal / confidential / restricted)
- [ ] PII handling procedures defined
- [ ] Data retention policy documented
- [ ] Data at rest encrypted (database encryption)
- [ ] Data in transit encrypted (TLS/SSL)

## Access Control

- [ ] Authentication mechanism configured
- [ ] Authorization roles defined (admin, operator, reader)
- [ ] API key or token management in place
- [ ] Principle of least privilege applied
- [ ] Service account credentials secured

## Secrets Management

- [ ] No hardcoded credentials in source code
- [ ] Environment variables used for secrets
- [ ] `.env` files excluded from version control
- [ ] Secret rotation policy defined
- [ ] Secrets stored in a vault or secure store

## Network Security

- [ ] Database not exposed to public network
- [ ] API endpoints protected (rate limiting, authentication)
- [ ] Firewall rules configured
- [ ] CORS policy defined (if web-facing)
- [ ] Internal services use private networking

## Code Security

- [ ] Dependencies scanned for vulnerabilities
- [ ] Input validation on all external data
- [ ] SQL injection prevention (SDK methods only, no raw SQL)
- [ ] Error messages do not leak internal details
- [ ] Logging does not include sensitive data

## Operational Security

- [ ] Audit logging enabled
- [ ] Security monitoring configured
- [ ] Incident response plan documented
- [ ] Backup encryption verified
- [ ] Recovery procedures tested

## Compliance (if applicable)

- [ ] Regulatory requirements identified (GDPR, HIPAA, SOC2, etc.)
- [ ] Data processing agreements in place
- [ ] Right to erasure / deletion procedures defined
- [ ] Consent management configured
- [ ] Compliance audit trail maintained
