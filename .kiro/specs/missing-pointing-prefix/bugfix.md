# Bugfix Requirements Document

## Introduction

The onboarding system has a question-ownership architecture conflict between two components that both attempt to ask closing 👉 questions. The `ask-bootcamper` hook (fires on every `agentStop`) is designed to be the single owner of all closing 👉 questions — it generates contextual questions based on what the agent just accomplished. However, `onboarding-flow.md` contains inline 👉 questions with WAIT instructions at the end of each step (e.g., "👉 Does this outline make sense?" WAIT, "👉 Which language would you like to use?" WAIT, "👉 Which track sounds right for you?" WAIT). This dual-source design creates three problems: (1) the agent may drop the 👉 prefix when paraphrasing inline questions because they're embedded in prose rather than generated by the hook's structured prompt, (2) the hook's "do nothing" suppression path — which relies on the LLM detecting an existing trailing 👉 question — is unreliable because prompt-based suppression is not guaranteed, and (3) when suppression fails, the bootcamper sees duplicate back-to-back questions. The correct architecture is: the agent presents information and stops, then the hook fires and asks a single contextual 👉 question. The `agent-instructions.md` closing-question ownership rule already mandates this ("Never end your turn with a closing question — the `ask-bootcamper` hook owns all closing questions"), but `onboarding-flow.md` contradicts it with inline closing questions and WAITs.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN the agent executes any onboarding step in onboarding-flow.md that ends with an inline 👉 question and WAIT instruction THEN the system has two competing sources of closing questions — the inline question from the steering file and the ask-bootcamper hook — creating a question-ownership conflict

1.2 WHEN the agent paraphrases an inline 👉 question from onboarding-flow.md THEN the system may drop the 👉 prefix because the question is embedded in steering file prose rather than being generated by the hook's structured prompt, which consistently applies the prefix

1.3 WHEN the agent's turn ends with an inline 👉 question from onboarding-flow.md AND the ask-bootcamper hook fires on agentStop THEN the hook may generate a second 👉 question because prompt-based suppression ("If your previous output already ends with a 👉 question, do nothing") is not reliably followed by LLMs, resulting in duplicate back-to-back questions

### Expected Behavior (Correct)

2.1 WHEN the agent executes any onboarding step in onboarding-flow.md THEN the step SHALL NOT contain inline 👉 closing questions or WAIT instructions; the agent SHALL present the step's information and then stop, allowing the ask-bootcamper hook to be the single owner of the closing 👉 question

2.2 WHEN the ask-bootcamper hook fires after the agent completes an onboarding step THEN the hook SHALL be the single source that generates a contextual 👉 question based on what the agent just presented, eliminating the possibility of dropped prefixes from paraphrased inline questions

2.3 WHEN onboarding-flow.md describes each step THEN the step SHALL present its information clearly enough that the ask-bootcamper hook can generate an appropriate contextual 👉 question without needing inline question text in the steering file

### Unchanged Behavior (Regression Prevention)

3.1 WHEN the ask-bootcamper hook fires on agentStop THEN the system SHALL CONTINUE TO generate contextual 👉 questions based on what the agent just accomplished

3.2 WHEN the ask-bootcamper hook fires on a no-op turn where no files changed and no substantive work was done THEN the system SHALL CONTINUE TO skip the recap and ask a contextual 👉 question

3.3 WHEN the agent-instructions.md closing-question ownership rule is evaluated THEN the system SHALL CONTINUE TO enforce that the ask-bootcamper hook owns all closing questions

3.4 WHEN the agent presents informational content (module overviews, status updates, file listings) that does not require user input THEN the system SHALL CONTINUE TO omit the 👉 prefix from that content

3.5 WHEN the agent executes the onboarding flow THEN the system SHALL CONTINUE TO follow the same step sequence (setup → language → prerequisites → introduction → track selection)
